Scatter variable-sized segments across ranks using MPI_Scatterv.

Distributes variable-sized segments of an integer array across ranks using MPI_Scatterv.

The caller is responsible for providing the source data array (src_values) on rank 0, as well as the counts and displacements arrays. The function simply scatters the data using MPI_Scatterv, and each rank receives local_n integers into local_buf.

This function does not generate or allocate the source buffer; it only performs the scatter. The caller must ensure that src_values is valid and populated on rank 0.

After scattering, the receive buffer (local_buf) is returned for convenience.

Parameters
src_valuessource buffer on rank 0; ignored on other ranks. This should be set to NULL if the caller does not intend to pass a pre-allocated buffer containing the data distribution to process.
local_bufCaller-provided receive buffer.
countsPer-rank receive counts (length comm_size) passed to MPI_Scatterv.
displsDisplacements matching counts.
local_nNumber of integers expected locally (recvcount).
rankRank of the calling process.
buf_sizeTotal number of integers available on rank 0. Used when use_src is false.
commCommunicator used for the scatter.
Returns
local_buf for chaining.

void tree_reduce ( struct QDigest * q,
int comm_size,
int rank,
MPI_Comm comm )

Performs a power-of-two tree reduction over distributed QDigests.

The routine first trims the communicator to the largest power of two not exceeding comm_size. Ranks in the prefixed [0, 2*orphans) window are paired (odd sends to the preceding even rank) and their digests are merged, after which odd senders exit the algorithm. Survivors form a compact communicator via MPI_Comm_split() so their ranks become dense in [0, p2).

The remaining ranks execute a log2(tree_size)-step binary-tree reduction:

  • On each level k, ranks with tree_rank % (2 * step_size) != 0 serialize their digest via to_string() and send it (size first, then payload) to the partner tree_rank - step_size.
  • Receivers collect the serialized digest, reconstruct it with from_string(), merge it into their local digest via merge(), and continue to the next level.
  • Senders drop out after transmitting.
Parameters
qPointer to the local QDigest that participates in the reduction.
comm_sizeTotal number of ranks in the input communicator.
rankRank of the calling process within the input communicator.
commInput communicator that contains all participants (e.g., MPI_COMM_WORLD).
Note
Serialization uses the current get_num_of_bytes()/to_string() pair, which assumes fixed field separators; once configurable delimiters are supported those literals will be replaced with runtime values.
All MPI messages use tag 0 and blocking send/recv; future work may introduce configurable tags or non-blocking operations.
